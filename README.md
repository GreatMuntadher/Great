# رحلة التعافي — Recovery Journey

تطبيق ويب شخصي لمتابعة رحلة التعافي اليومية — سجّل أيامك، قاوم الرغبات، وابنِ نفسك يوماً بيوم.

---

## المميزات الرئيسية

- **عداد الأيام النظيفة** — يتابع سلسلة التعافي اليومية وأفضل سلسلة محققة
- **خطة اليوم** — 10 مهام يومية قابلة للتفعيل مع شريط تقدم
- **بروتوكول الطوارئ (SOS)** — 4 مراحل لمواجهة الرغبات الحادة فوراً
- **تقييم اليوم** — 20 سؤالاً على 6 فئات تحسب درجة التقدم اليومي ومستوى الخطر
- **التحديات** — إنشاء تحديات شخصية (30/90 يوم أو مخصص) مع شبكة أيام بصرية
- **الكالندر** — عرض شهري ملوّن حسب الأداء مع تفاصيل كل يوم عند الضغط
- **التقييمات** — 7 مؤشرات أداء برسوم SVG متموجة وفلترة زمنية
- **الإحصائيات** — مركز تحليل شامل: موجة الأداء، تحليل التحديات، ملخص ذكي
- **السجل اليومي** — كتابة حرة مع حفظ لكل يوم
- **المذكرات** — تقييم الجلسة مع أسئلة مخصصة وأرشيف قابل للبحث
- **المحفزات** — خريطة الرغبات مع خطط لكل محفز وبدائل سريعة
- **التقارير الذكية** — تقرير أسبوعي تلقائي بـ 6 مقاييس وتوصية
- **المراجعة الأسبوعية** — 4 أسئلة انعكاسية أسبوعية
- **وضع الإغلاق** — حماية مؤقتة بعد الانتكاسة
- **الخصوصية** — إخفاء سريع + قفل PIN
- **النسخ الاحتياطي** — تصدير/استيراد JSON كامل

## التقنيات المستخدمة

- **HTML5 / CSS3 / JavaScript ES6+** — بدون أي framework أو مكتبة خارجية
- **SPA** — تطبيق صفحة واحدة بتنقل عبر `data-sec` + `go()`
- **RTL عربي كامل** — `dir="rtl"` + `lang="ar"` + خط Cairo
- **localStorage** — كل البيانات محفوظة محلياً، بدون backend أو cloud
- **SVG** — رسوم الأداء المتموجة (cubic bezier) بدون Chart.js أو Three.js
- **PWA** — Service Worker + manifest.json + offline support
- **Glassmorphism + Space Theme** — تصميم فضائي داكن مع تأثيرات CSS فقط

## طريقة التشغيل محلياً

### الطريقة السريعة (خادم بسيط):

```bash
cd recovery-journey
npx serve -l 3000 ..
```

ثم افتح: `http://localhost:3000/recovery-journey/`

### أو مع Python:

```bash
cd recovery-journey/..
python -m http.server 3000
```

ثم افتح: `http://localhost:3000/recovery-journey/`

> **مهم:** يجب تشغيل الموقع من خادم HTTP (وليس فتح `index.html` مباشرة) لكي يعمل Service Worker.

---

## طريقة النشر على GitHub Pages

1. ارفع المشروع على GitHub في مستودع باسم مثلاً `recovery-journey`
2. اذهب إلى **Settings → Pages**
3. اختر **Source: Deploy from a branch**
4. اختر الفرع `main` والمجلد `/recovery-journey`
5. بعد النشر، الموقع يكون على: `https://username.github.io/recovery-journey/`

> تأكد أن `start_url` و `scope` في `manifest.json` تطابق مسار الموقع.

## ملاحظات PWA

- بعد أول زيارة، الموقع يعمل **بدون إنترنت** من الكاش
- يمكن تثبيته على الهاتف أو الحاسوب من المتصفح
- **Cache الحالي:** `recovery-journey-v10`
- الأيقونة: SVG مخصصة في `assets/icons/icon.svg`
- عند تحديث الكود: ارفع رقم `CACHE_NAME` في `sw.js` لمسح الكاش القديم

---

## ملاحظات النسخ الاحتياطي

- **تصدير:** ينتج ملف JSON يحتوي على جميع مفاتيح localStorage
- **استيراد:** يتحقق من صحة الملف قبل الاستيراد ويرفض الملفات غير الصالحة
- **الملفات المدعومة:** `.json` فقط
- يُنصح بالتصدير الدوري وحفظ النسخة في مكان آمن
